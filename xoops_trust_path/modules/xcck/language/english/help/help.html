Overview

xCCK, an acronym for XOOPSCube Contents Creation Kit, is a simple yet powerful and versatile no-code database module. A Contents Creation Kit (CCK) empowers users to create and manage various content types without requiring programming expertise.

This module fully leverages XOOPSCube's capabilities by integrating with common module groups nd features such as parent-child pages, sub-table configurations, categories, tags, images, and workflows. This integration allows for highly customizable functionalities and design.

xCCK enables the flexible creation of basic web applications for storing and displaying data, suitable for static content, news, product catalogs, image albums, blogs, link lists, and more.

xCCK was initially distributed at OSC2011 Nagoya.
The information above is based on the xCCK introduction presentation held on September 16, 2011, at the Ebisu co-working space.

Get started

To get started with xCCK, follow these steps:

1. Installation

Download the xCCK module from the XOOPSCube repository.

Upload the module to your installation directory.

Install the module through the administration interface.

Administrators can add data to pages. Choose field types like string, text, int, date, category, select, or checkbox.

Parent Page - Child Page Settings

Create child pages and organize them into a tree structure.

Set subpage for page

Link multiple xCCKs to set subpages. Control product size options with subpages.

Category or group-based permission settings

Use Legacy_Category and Legacy_Group modules for detailed permissions.

Classification by tags

Classify pages by tags using the Legacy_Tag module.

Image attachment

Attach images to pages with the Legacy_Image module.

Workflow

Apply approval functions for page publication with Legacy_Workflow.

Practical Implementation Examples

xCCK can be used as a news module. It can also be implemented as a product module. Furthermore, xCCK can serve as an inquiry module. The module also includes the functionality to add input value validation.

Definitions

The definition class in the xCCK module is responsible for handling the editing of definitions within the module's graphical user interface (GUI). Here's a breakdown of what this class allows users to do:

1. Handler Retrieval

The class provides a method to retrieve the handler for the definition objects, which is essential for performing operations on these objects.

2. Page Title Management

It includes functionality to get the page title based on the field name of the object being edited, which helps in displaying the correct title in the GUI.

3. Permission Checking

The class checks if the current user has the necessary permissions to edit the definition. This ensures that only authorized users can make changes.

4. Preparation

It prepares the environment for editing by setting up necessary components like the category manager.

5. Action Form Setup

The class sets up the action form used in the editing process, ensuring that the form is correctly initialized and ready for user input.

6. Header Script Management

It manages JavaScript that controls the behavior of the form fields in the GUI, such as enabling or disabling fields based on user interactions.

7. Duplicate Check

Before saving changes, the class checks for duplicate definitions to prevent conflicts and ensure data integrity.

8. View Execution

The class provides methods to handle different outcomes of the editing process, such as successful edits, errors, or cancellations. These methods determine what the user sees next in the GUI.

The Definition feature facilitates the editing of definitions in the xCCK module, providing a structured and secure way for users to modify content definitions through the GUI.

Page

The Page feature in the xCCK module is designed to facilitate the editing of pages within the module's graphical user interface (GUI). Here's what this class allows users to do:

1. Page Identification

It retrieves the ID of the page being edited, accommodating single post configurations.

2. Category Management

The class determines the category ID associated with the page, which is crucial for organizing content and managing permissions.

3. Parent-Child Relationships

It identifies the parent ID of the page, enabling hierarchical structuring of pages.

4. Handler Setup

The class sets up handlers for page objects, allowing for operations like creation and retrieval of page data.

5. Title Management

It retrieves the title of the page for display purposes in the GUI.

6. Permission Checking

The class checks if the user has the necessary permissions to edit the page, ensuring that only authorized users can make changes.

7. Action Form Setup

It sets up the action form used in the editing process, preparing it for user input.

8. Object Preparation

The class prepares the page object for editing, including setting default statuses and loading paths and tags.

9. Attribute Setup

It configures various attributes for rendering, such as category trees, field definitions, and image setups.

10. Preview and Execution

The class handles previewing changes and executing updates, including saving workflows and updating files.

11. Header Script Management

It manages JavaScript for the GUI, such as date picker scripts and map editing scripts.

The Page feature provides a comprehensive framework for editing pages in the xCCK module, offering features that support content management, permission control, and user interaction within the GUI.

Revision

The Revision feature in xCCK module is designed to manage the editing of revisions within the module's graphical user interface (GUI):

1. Revision Identification

It retrieves the ID of the revision being edited, ensuring that the correct revision is accessed for modifications.

2. Main Table Association

The class determines the main table ID associated with the revision, which is crucial for maintaining relationships between main and revision entries.

3. Category Management

It identifies the category ID for the revision, which helps in organizing content and managing permissions.

4. Parent-Child Relationships

The class retrieves the parent ID of the revision, allowing for hierarchical structuring of revisions.

5. Handler Setup

It sets up handlers for revision objects, enabling operations like creation and retrieval of revision data.

6. Title Management

The class retrieves the title of the revision for display purposes in the GUI.

7. Permission Checking

It checks if the user has the necessary permissions to edit the revision, ensuring that only authorized users can make changes.

8. Action Form Setup

The class sets up the action form used in the editing process, preparing it for user input.

9. Object Preparation

It prepares the revision object for editing, including setting default statuses and loading related data.

10. Attribute Setup

The class configures various attributes for rendering, such as category trees, field definitions, and image setups.

11. Preview and Execution

It handles previewing changes and executing updates, including saving workflows and updating files.

12. Header Script Management

The class manages JavaScript for the GUI, such as date picker scripts and map editing scripts.

13. Revision Viewing

The class allows users to view revisions of a specific item, providing a historical record of changes made to the item.

14. Revision Filtering

The class allows users to filter revisions based on various criteria, such as date range, author, or content changes.

15. Revision Sorting

The class allows users to sort revisions based on different criteria, such as date, author, or content changes.

16. Revision Pagination

The class provides pagination functionality for displaying revisions in manageable chunks, allowing users to navigate through multiple pages of revisions.

17. Revision Search

The class allows users to search for specific revisions based on keywords or criteria.

18. Revision Export

The class provides functionality for exporting revision data in various formats, such as CSV or PDF.

19. Revision Import

The class provides functionality for importing revision data from external sources, such as CSV or PDF files.

20. Revision Comparison

The class allows users to compare revisions side by side, highlighting differences and providing a visual representation of changes.

21. Revision History

The class provides a history of revisions for a specific item, showing the changes made to the item over time.

22. Revision Approval

The class allows users to approve or reject revisions, controlling the visibility and publication status of the item.

23. Revision Comments

The class provides a commenting system for revisions, allowing users to provide feedback or discuss changes made to the item.

24. Revision Notifications

The class sends notifications to users when a new revision is created or when a revision is approved or rejected.

25. Revision Permissions

The class controls access to revision data based on user permissions, ensuring that only authorized users can view or modify revision data.

26. Revision Statistics

The class provides statistics and analytics related to revision data, such as the number of revisions created, the average time between revisions, and the most active users.

27. Revision Workflows

The class integrates with workflow management systems, allowing users to track the status of revisions and apply workflows to them.

28. Revision Versioning

The class enables versioning of revisions, allowing users to track changes made to the item over time.

29. Revision Collaboration

The class enables collaboration among users, allowing multiple users to work on the same item simultaneously.

Revision provides a comprehensive framework for editing revisions in the xCCK module, offering features that support content management, permission control, and user interaction within the GUI. This allows you to view and interact with revision data in a structured and categorized manner within the application's GUI. It provides functionalities for filtering, organizing, and displaying revision information based on the application's configuration settings.
