xCCK is a kind of general-purpose database module, which is an acronym for XOOPS Cube Contents Creation Kit.

It is a new generation universal module that fully utilizes the functions of XOOPS Cube Legacy 2.2. You can freely design items as much as you like, and by linking with common module groups such as parent-child pages, sub-table settings, and categories, even those who cannot program can realize desired functions.

You can flexibly create basic web applications that store and display data, and can be used for static content, news, product catalogs, image albums, blogs, links, and more.

    Main function

    >
    Site administrator freely defines items

    The administrator can freely increase the data held by the page. Select field type from string (single line of text), text (multiple lines of text), int (integer), date (select date), category, select, checkbox can do.

    Parent Page - Child Page Settings

    You can create child pages for a page and organize the pages into a tree structure.

    Set subpage for page

    By linking multiple xCCKs, you can set subpages for a page. For example, you can control what size options are available for a product with subpage functionality.

    Category or group-based permission settings

    By using a common module that conforms to Legacy_Category and Legacy_Group, you can set detailed permissions for viewing, editing, and management.

    Classification by tags

    By using a common module that conforms to Legacy_Tag, pages can be classified by tags.

    image attached

    You can attach an image to a page by using a common module that conforms to Legacy_Image.

    workflow

    By using a module that conforms to Legacy_Workflow, you can apply the approval function for page publication.

    • leimg (image management)
    • letag (tag management)
    • lefeed (Manage new arrivals)
    • leprogress (workflow management)
    • *lecat is in XCL2.2 extras/extra_modules.
